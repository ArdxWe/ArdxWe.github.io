RISC-V

RISC-V 是一个基于 RISC 的开源指令集架构

ISA

基本 RISC-V ISA：

  • RV32I： 基本整数指令集， 32 bits
  • RV64I： 基本整数指令集， 64 bits
  • RV32E： 基本整数指令集， 32 bits, 16 registers
  • RV128I: 基本整数指令集， 128 bits

扩展：

  • F: 单精度浮点扩展
  • D： 双精度浮点扩展

Zfinx

  • 将所有的浮点寄存器扩展的 浮点寄存器 f 改为 整数寄存器 x
  • 和之前相比， 仅汇编语法变化， 对应二进制码不变

QEMU

  • Quick Emulator 的缩写
  • 开源虚拟化软件
  • 可以模拟一些特定的处理器架构， 通过 TCG(Tiny Code Generator) 动态转换机器代码

原理

  • TARGET: QEMU 所模拟的处理器架构
  • HOST： QEMU 所真正运行所在的机器架构
  • TCG： target 代码的动态转换执行者

QEMU 通过将 GUEST CODE 转化为 HOST CODE来进行模拟， 转换工作分为两部分：

  • TARGET 代码块 ——-> TCG ops(平台独立的中间符号)
  • TCG ops ———> Host code

动态转换

也就是说：

  • 以 gen 起始的函数用来生成 TCG ops
  • 以 tcg_gen 起始的函数用来生成 Host code

举个例子:

此函数所在位置： qemu/target/riscv/insn_trans/trans_rvf.c.inc

  • gen_get_gpr: 读 rs1 寄存器到 t0
  • tcg_gen_addi_tl: t0 加 立即数 到 t0
  • tcg_gen_qemu_ld_i64: 寻址 t0 到 目的寄存器

源码结构

文件夹 作用
/hw/ 模拟硬件
/target/xyz/ 被模拟的 xyz 体系结构代码块转换为 TCG ops 的具体实现(TCG 前端)
/TCG/xyz/ TCG ops 转化为运行 QEMU 的体系架构的机器码的具体实现(TCG 后端)

在 QEMU 中如何实现 Zfinx

RISCV ZFinx

  • 当 QEMU 检测到 guest code 实现 Zfinx 扩展时， 替换浮点寄存器为整数寄存器(似乎生成的后端机器码所操作的寄存器只是内存指定位置)
  • remove all floating point load (e.g. FLW) store (e.g. FSW) and register move (e.g. FMV.X.W) encodings as the integer versions of these are required instead
  • 更改 NaN-boxing
  • 使用特定的 浮点指令进行测试 测试参考
