如何维护当前所有作业状态

我定义了一个结构体 job_t ，保存

  • 当前作业 pid
  • 作业的 id
  • 当前作业状态
  • 执行此进程的命令

读入命令时，程序会先分析前台还是后台(简单的判断最后一个字符是否为 &)

父进程如何等待前台进程执行完

每次 fork 时，父进程记录当前子进程 pid

当 prevpid 不为 0, 一直阻塞

当父进程 SIGCHLD 处理函数发现是子进程发来的信号，置为0, 从而解除阻塞，继续读取命令

IO 如何在执行一条命令重定向

得到命令字符串时，传给 ioredirection 函数处理

若存在 > ，打开下一个字符串指定的文件重定向到 stdout

若存在 < ,打开下一个字符串指定的文件重定向到 stdin
